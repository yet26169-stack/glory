════════════════════════════════════════════════════════════════════════════════

SINGLE-THREADED DESIGN: All command recording occurs in drawFrame() (main thread)

Frame Synchronization (Renderer.cpp:255-350):
  1. vkWaitForFences(inFlightFence)        ← GPU⇒CPU: wait oldest frame
  2. vkAcquireNextImageKHR(swapchain)      ← Get swapchain image index

  5. vkQueueSubmit(cmd, imgSem→renderSem, inFlightFence)

8. RENDER PIPELINE TIMELINE
════════════════════════════════════════════════════════════════════════════════

Per-Frame Execution (60 FPS target):

T=0ms:  glfwPollEvents()
T=1ms:  drawFrame() called
        ├─ Update game systems (abilities, physics, AI)
        ├─ vfxRenderer.processQueue()    [SPSC event drain from game thread]
        ├─ Update particle emitters
        ├─ Update trails, effects, decals
        └─ dt capped at 50ms (20 FPS floor to prevent stutter)

T=2ms:  GPU Synchronization
        ├─ vkWaitForFences(frame[n-2])   [Wait for n-2 frame to complete]
        ├─ vkAcquireNextImageKHR()       [Get swapchain image index]
        └─ Ready to record command buffer

T=3ms:  Record Command Buffer (vkBeginCommandBuffer → vkEndCommandBuffer)
        ├─ Update per-frame UBOs (view, proj, light, fog)
        ├─ Particle compute dispatch
        ├─ Opaque render pass (models, grid, decals)
        ├─ Transparent render pass (particles, effects)
        ├─ Bloom extraction & blur (compute)
        ├─ Tone-map to swapchain
        └─ ImGui overlay

T=4ms:  Queue Submission & GPU Execution
        ├─ vkResetFences(frame[n])
        ├─ vkQueueSubmit(imageAvailableSemaphore → renderFinishedSemaphore)
        ├─ GPU begins execution (asynchronously)
        └─ CPU can proceed to next frame

T=5ms:  Present to Screen
        ├─ vkPresentKHR(renderFinishedSemaphore)
        └─ Image displayed at next vertical sync

T=16ms: Next Frame (assuming 60 FPS, 16.67ms per frame)
        (GPU executing frame[n], CPU recording frame[n+1])

9. KEY ARCHITECTURAL DECISIONS & TRADEOFFS
════════════════════════════════════════════════════════════════════════════════

Decision                          Rationale                    Tradeoff
─────────────────────────────────────────────────────────────────────────────
Single-threaded command           Simplicity, no              No parallelized draw
recording                         synchronization overhead    call recording

VFX SPSC queue (lock-free)        Zero-copy, minimal          Game⇒Render is
                                  latency between threads     asynchronous (1-frame
                                                              delay)

Persistent UBO/SSBO mapping       Avoid CPU→GPU round-trip,   Must use explicit
                                  cache locality              flush()

Bindless textures (64 max)        Material reuse, reduced     Fixed array size limit
                                  descriptor updates          (extend via
                                                              specialization constants)

Dedicated transfer queue          DMA offload on discrete     Falls back to graphics
preference                        GPUs (NVIDIA, AMD)          queue if unavailable

Compute-outside-renderpass        Avoid render pass           Extra command buffer
                                  incompatibility             complexity

Two render passes                 Layered rendering (opaque   RenderPass object
(HDR + LOAD)                      then transparent)           complexity

Fixed MAX_INSTANCES=256           Simple CPU→GPU data         Limits entities per
                                  transfer                    frame (~50-100 actual)

Wireframe pipeline variant        Debug visualization         Slight GPU footprint
                                                              overhead
